Research

LLM-Based Program Reduction

Undergraduate research with Professor Arpit evaluating Gemini's ability to reduce Java test methods using the Defects4J benchmark suite (30 bugs). Methodology uses a single prompt per method with no error message, classifying results as TN / TP / FP / FN against a hand-reduced ground truth. Key finding: Gemini explicitly requested error messages to identify failing assertions, raising a methodological question about prompt design in LLM-assisted debugging.

Python Gemini API Defects4J Java LLM Evaluation
